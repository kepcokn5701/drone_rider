# Drone Rider MVP 프로토타입 계획 v0.3

> **목표**: 12주(3개월) 안에 "공중판 카트라이더 느낌" + "결함 점검 메커닉 교육 효과" + "전사 표준 PC 호환" 셋 다 검증할 수 있는 최소 단위 웹 게임을 만든다. **MVP 도메인은 배전선로(22.9kV)**.

**상위 문서**: [CONCEPT.md](./CONCEPT.md) (v0.4)
**참조 자료**:
- 송전(Phase 3용): `드론 설비점검방법 및 점검양식/첨부4_드론 활용 OPGW 촬영 가이드_수합본.pdf`
- 배전(MVP): 한전 사내 매뉴얼 확보 예정

> **v0.1 → v0.2 → v0.3 변경**:
> - v0.2: UE5 → Babylon.js 웹 스택
> - **v0.3: 게임명 Drone Rider, MVP 도메인 송전→배전. 자산은 Poly Haven `modular_electricity_poles` 활용.**

## 0. 한 페이지 요약

```
What  : 1조형 배전 전주 6~8개가 늘어선 3km 가상 코스를
        AI 봇 1대와 경쟁하며 비행, 각 전주에서 점검 게이트 통과,
        시간 + 정확도 종합 점수 산출. 웹 브라우저로 플레이.
How   : Babylon.js + Vite + TypeScript. Vercel 호스팅.
        자산은 Poly Haven CC0 (modular_electricity_poles).
Who   : 1~2인 개발
When  : 12주, 마일스톤 10개
Why   : 게임 재미 + 교육 효과 + 전사 표준 PC 호환성 셋 다 검증
        Phase 2(멀티 + 결함 추가), Phase 3(송전선로 모드)로 확장
```

## 2. 스코프 (in / out)

### IN (MVP 범위)
- 가상 평지 배전선로 1개, 길이 약 3km
- 전주 6~8개 (`modular_electricity_poles` 활용)
- 결함 3종: **애자 균열·오손 / 변압기 누유 / 완금 부식**
- AI 봇 1대 + 러버밴딩
- 환경 이벤트 1종(전자파 간섭) + 돌풍 1종(옵션)
- 모드 3개: 튜토리얼, 챔피언십(봇과 1:1), 타임어택
- 시야: 3인칭 + FPV 줌
- 점수: 시간 + 점검 정확도 + 부스터 콤보
- 입력: 키보드 + Gamepad API
- 배포: Vercel 인터넷망, 사내 PC에서 URL 접속

### OUT (Phase 2 이후)
- 멀티플레이
- 송전선로 모드 (OPGW, 154kV+ 격자 송전탑) → **Phase 3**
- 실제 사업소 지형 / DJI Terra 임포트
- 맵 에디터
- 추가 결함 종류 (COS, 피뢰기, 가공지선 등)
- 모바일/태블릿 빌드
- 사운드/BGM 본격 작곡 (효과음만 임시)
- AAA급 비주얼 — Quixel-style 디테일은 욕심 X
- 인트라넷 이전 (반응 보고 Phase 2)

## 4. 마일스톤 상세

### M1. Node.js + Vite + Babylon.js (W1) — ✓ 완료

✓ Node.js v24, Vite 8, Babylon.js core/loaders/gui/materials 설치
✓ TypeScript 베이스 + canvas + scene + 카메라 + 박스 드론
✓ 키보드 입력 + 카메라 추적
✓ 보너스: gradient sky shader, CellMaterial 카툰 셰이딩, FXAA + Bloom + Tone Mapping, 잔디 텍스처, glTF 모델 로드

### M2. 아케이드 드론 컨트롤러 ★ 핵심 (W2)

박스 드론이 **카트라이더처럼** 비행. 시뮬 X, 아케이드 ⭕.

**기술 노트**
- 자체 `DroneController` 클래스 (Babylon.js Mesh wrapping)
- `scene.onBeforeRenderObservable`에서 매 프레임 물리 업데이트
- 입력: KeyboardEventTypes + Gamepad API
- 부스터 게이지 + 가속 배율 + 드리프트 회전 반경 축소

**튜닝 파라미터 (1차)**

| 파라미터 | 값 |
|---|---|
| 최대 속도 | 40 m/s (~144 km/h) |
| 가속 시간 (0→max) | 2.5s |
| 부스터 배율 | 1.5x |
| 부스터 지속 | 2.0s |
| 부스터 게이지 회복 | 드리프트 1초당 +20% |
| 회전 속도 (일반) | 60°/s |
| 회전 속도 (드리프트) | 120°/s |
| 자동 고도 | y = 30 lock |

**DoD**
- 1분 자유 비행 후 "어, 이거 재밌네"
- 다른 사람 1명, 5분 안에 자유 비행
- 멀미 없음 (visual roll 살짝)
- HD 530 PC에서 60 FPS 유지

### M4. 전주 + 충돌 + 점검 게이트 (W4)

- `modular_electricity_poles` 모델 정렬 배치 (혹은 메인 전주만 추출)
- 충돌: 1초 스턴 + 약간 후진
- 각 전주마다 점검 게이트 트리거 → 진입 시 호버링 + 카메라 보간 FPV

**점검 게이트 시퀀스 (배전 — MVP 압축)**
```
[게이트 진입] 호버 + FPV 줌인
  ① 애자 카드   (전주 상단)
  ② 변압기 카드 (전주 중단)
  ③ 완금 카드  (가로대)
[게이트 이탈] 부스터 보상 + 3인칭 복귀
```

각 카드 제한시간 2초. 합쳐 ~7~8초 호버.

**DoD**: 게이트 진입·이탈 동선 자연스러움. 더미 카드로 흐름 검증.

### M5. 결함 카드 UI + 정답 판정 (W5)

**UI**: Babylon.js GUI 풀스크린 오버레이

```
┌─────────────────────────┐
│  애자 점검               │
│  ⏱ 2.0  ───────         │
│  [드론 시점 화면]         │
│  결함이 있나요?           │
│  [1] 정상  [2] 결함       │
└─────────────────────────┘
```

**정답 판정**
- 결함 인스턴스: `{타입, 결함_여부, 결함_심각도, 결함_종류}`
- 정답: 부스터 +50%, 콤보 +1, 점수 +기본×(1+콤보×0.1)
- 오답: 시간 -2초, 콤보 리셋

**MVP 결함 3종**
- 애자: 정상 vs 균열·오손 (선형 균열 또는 검은 침착)
- 변압기: 정상 vs 누유 (외함에 검은 기름 자국)
- 완금: 정상 vs 부식 (녹·변색)

**데이터**: `src/data/defect_catalog.json`

```json
{
  "defects": [
    {
      "id": "insulator_crack_dirt",
      "fitting": "애자",
      "detection": ["균열", "오손"],
      "visual": { "param": "surfaceWear", "normal": [0,0.2], "defect": [0.6,0.9] }
    },
    {
      "id": "transformer_oil_leak",
      "fitting": "주상변압기",
      "detection": "누유",
      "visual": { "param": "oilStain", "normal": false, "defect": true }
    },
    {
      "id": "crossarm_corrosion",
      "fitting": "완금",
      "detection": "부식",
      "visual": { "param": "rustAmount", "normal": [0,0.15], "defect": [0.5,0.9] }
    }
  ]
}
```

**DoD**: 3종 카드 + 정답/오답 + 점수 가산. 실제 결함 시각화는 M9.

### M6. 점수 + 부스터/드리프트 + 페널티 (W6)

**점수 수식**
```
완주점수 = 시간점수 + 점검점수 + 콤보보너스 - 페널티
시간점수    = max(0, (180 - 완주시간_초) × 50)
점검점수    = ΣΣ (정답 +1000, 오답 -500, miss -1500)
콤보보너스  = 연속정답수^1.5 × 200
페널티      = 충돌수 × 500 + 추락수 × 2000

등급: S(>20000), A(>15000), B(>10000), C(>5000), Fail
```

**페널티**
- 전주 충돌: 1초 스턴 + 후진 5m + 게이지 -50%
- 전선 접촉: 0.5초 컨트롤 마비
- 추락: 5초 후 마지막 체크포인트

### M9. 배전 자산 정리 + 결함 3종 시각 구현 (W9-10)

**자산 정리**
- `modular_electricity_poles` 모델 분석: 메쉬 구조·메인 전주·부속들 식별
- 옵션 A: 키트 통째 (한 인스턴스 = "한 배전 구간")
- 옵션 B: 메인 전주만 추출 + 자체 변압기·완금 배치 (정렬 깔끔)
- 시연 효과·교육 효과 기준으로 선택

**결함 3종 시각**

| 결함 | 게임 내 시각 | 정상 | 결함 |
|---|---|---|---|
| 애자 균열·오손 | 머티리얼 surfaceWear 파라미터 | 0~0.2 | 0.6~0.9 |
| 변압기 누유 | 외함에 oil decal layer | 비활성 | 활성 |
| 완금 부식 | rust map blend | 0~0.15 | 0.5~0.9 |

각 게이트마다 50% 확률로 결함 1개 삽입 → 매번 다른 경험.

**DoD**: 베테랑이 봐도 구분 가능. HD 530 30 FPS 유지.
