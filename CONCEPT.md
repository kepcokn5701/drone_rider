# Drone Rider (드론라이더) — 컨셉 v0.4

> **한 줄 피치**: 카트라이더처럼 짜릿한 추월·부스터·이벤트가 있는 **공중판 아케이드 레이서**. 배전 전주 사이를 질주하며 점검을 빠르고 정확하게 해낼수록 옆 봇을 추월한다. 한전 배전·송전 점검 교육을 게임으로. 웹 브라우저로 즉시 플레이.

**장르**: 아케이드 레이싱 + 교육 시뮬 (Edutainment)
**플랫폼**: 웹 브라우저 (전사 표준 PC 호환 — Intel HD 530 + 8GB RAM 작동)
**엔진**: Babylon.js + WebGL 2.0
**배포**: 1차 인터넷망(Vercel) → 반응 보고 인트라넷 이전 검토
**도메인 우선순위**: **배전선로(MVP) → 송전선로(Phase 3)**

> **변경 이력 요약**:
> - v0.1: 드론 시뮬 + 점검 종합점수 초안
> - v0.2: 사실적 시뮬 → 카트라이더식 아케이드, 게임명 "송전라이더"
> - v0.3: UE5 → Babylon.js 웹 스택 (사내 PC 사양 + ICT 정책 제약)
> - **v0.4: 게임명 "Drone Rider"로 변경 + 도메인 우선순위 송전→배전 (Poly Haven 자산 핏 + 한전 내부 비중)**

## 1. 비전

한전 배전 점검 담당자가 **퇴근 후 한 판 돌려도 재밌고**, 신규자가 **한 시간 플레이 후 실제 결함 사진을 더 잘 식별**하는 게임. 인트라넷 URL 하나 받으면 누구나 즉시 시작. 1차로 배전선로 점검(가장 많은 직원·가장 잦은 사고)을 다루고, Phase 3에서 송전선로(154/345/765kV)로 확장.

**핵심 통찰**: 카트라이더가 운전 학원이 아니듯, 드론라이더도 점검 시뮬레이터가 아니라 **점검 의사결정의 본질(속도×정확도, 우선순위, 패턴 인식)을 게임 메커닉으로 압축한 아케이드**. 비주얼은 카툰풍이면 충분.

## 2. 차별점 (3축)

| 축 | MissionFlight PRO | Drone Rider |
|---|---|---|
| **장르** | 드론 비행 시뮬레이션 | **아케이드 레이싱** (카트라이더식) |
| **컨트롤** | 실제 드론 컨트롤러 모사 (학습곡선 ↑) | **방향 + 부스터 + 점검 1버튼** (즉시 플레이) |
| **콘텐츠** | 일반 송전탑·시나리오 | **한전 배전·송전 표준 + 매뉴얼 기반 결함 카탈로그** |

## 3. 핵심 게임플레이 루프 — "카트라이더 매핑"

### 카트라이더 → Drone Rider 메커닉 대응

| KartRider 요소 | Drone Rider 매핑 |
|---|---|
| 직진 / 좌우 | 공중 전진 / yaw + 살짝 roll |
| 드리프트 | 급선회 — 코너 통과 시 부스터 게이지 충전 |
| 부스터 | 일시 가속 (~2초) |
| 아이템 박스 | **전주 점검 게이트** (결함 카드 등장) |
| 자석·물폭탄·전기 | **전자파 간섭·돌풍·조류 떼** (환경 이벤트) |
| 트랙 | 배전선로 위 공중 코스 (체크포인트 게이트) |
| 옆 카트 추월 | **AI 봇 드론 추월** (싱글에서도 짜릿함) |

### 한 랩 흐름

```
출발 → 가속 → 코너에서 드리프트로 부스터 충전
     ↓
   전주 점검 게이트 진입 → 자동 호버링 (~1초)
     ↓
   결함 카드 등장 (3~5초 제한) → 분류 선택
     ↓
   정답 + 빠를수록 → 부스터 보상, 옆 봇 추월
   오답 / 시간 만료    → 봇한테 추월당함
     ↓
   다음 구간 → 환경 이벤트 발생 (전자파 간섭으로 1초 흔들림)
     ↓
   결승선 통과 → 종합 점수
```

## 4. 컨트롤 (카트라이더식 단순화)

| 입력 | 동작 |
|---|---|
| `W` / 위 | 전진 가속 |
| `A` / `D` | 좌우 (자동 roll + yaw) |
| `S` / 아래 | 감속 / 후진 |
| `Space` | 부스터 |
| `Shift` | 드리프트(급선회) — 부스터 충전 |
| `E` | 점검 액션 (게이트 진입 시 결함 카드 활성) |
| `1/2/3/4` | 결함 분류 선택 |
| `C` | 시야 전환 (3인칭 ↔ FPV 줌) |

Gamepad API로 Xbox/PS 컨트롤러 자동 인식. 키보드 우선.

## 5. 점수 시스템

```
총점 = 시간 점수 + 점검 정확도 점수 + 부스터 콤보 보너스 - 페널티

[시간 점수]      완주 시간 기반 (S/A/B/C 등급)
[점검 정확도]
  - True Positive  : 큰 +
  - True Negative  : 작은 +
  - False Negative : 큰 -   ★ 결함 miss = 가장 큰 감점
  - False Positive : 중간 - ★ 오신고 = 자원 낭비 페널티
[부스터 콤보]    점검 정답 빠르게(2초 내) → 부스터 자동 충전 + 콤보 ↑
[페널티]          전주 충돌 1~2초 스턴 / 전선 접촉 1초 컨트롤 마비 / 추락 체크포인트 복귀
```

**의도된 학습 효과**: False Negative 감점 비대칭으로 "의심되면 무조건 신고" 가 아닌 **"빠르고 정확하게"** 훈련.

## 8. 점검 콘텐츠 — 배전선로 (MVP)

### 8.1 배전선로 구조 (한전 표준)

한전 배전 = **22.9kV** 표준. 변전소 → 배전 전주 → 수용가.

**점검 대상 설비**:

| 분류 | 설비 | MVP 대상 |
|---|---|---|
| 지지물 | 전주 (콘크리트/목재) | ✓ |
| 지지물 | 완금 / 완철 (가로대) | ✓ |
| 절연 | 핀애자 / 현수애자 / LP애자 | ✓ |
| 절연 | COS (Cut-Out Switch, 컷아웃 스위치) | Phase 2 |
| 변환 | 주상변압기 (Pole Transformer) | ✓ |
| 보호 | 피뢰기 (Lightning Arrester) | Phase 2 |
| 보호 | 가공지선 (Ground wire) | Phase 2 |
| 도체 | ACSR 전선 (Aluminum Conductor) | Phase 2 |
| 부속 | 인입선, 슬리브, 데드엔드 클램프 | Phase 2 |

### 8.2 결함 카탈로그 (배전 일반 기준 — 추후 사내 매뉴얼로 정밀화)

| 설비 | 결함 종류 | 시각 패턴 |
|---|---|---|
| **애자** | 균열 / 오손 / 섬락 흔적 / 깨짐 | 선형 균열·검은 침착·탄화·결손 |
| **변압기** | 누유 / 외관 부식 / 외함 손상 | 기름 자국·녹·찌그러짐 |
| **완금/완철** | 부식 / 휨 / 볼트 풀림 | 녹·각도 변형 |
| **전주 본체** | 균열·기울기 / 콘크리트 박리 | 선형 균열·표면 박리 |
| **COS** | 동작 불량 (소체 끊김) / 외관 손상 | 부품 위치 이상 |
| **가공지선** | 단선 / 처짐 | 선 끊김·과처짐 |
| **피뢰기** | 외관 손상 / 접지선 풀림 | 결함 부분 |

### 8.3 MVP 결함 3종 선정 (시각 인지 패턴 3종 커버)

1. **애자 균열·오손** — 표면 패턴 인지 (선형/얼룩)
2. **변압기 누유** — 색·자국 인지 (검은 기름)
3. **완금 부식** — 색·텍스처 인지 (녹·변색)

→ "균열 / 얼룩 / 녹" 3가지 인지 회로를 한 번에 훈련.

> **추후**: 한전 배전 점검 매뉴얼이 확보되면 정확한 검출 기준·임계값으로 정밀화. 송전 OPGW 가이드처럼 게임 정답 판정 = 한전 자동분석 기준 호환 구조 유지.

## 12. MVP 정의 (3개월 목표)

**스코프 (작게)**
- 맵: **가상 평지 배전선로** 1개, 길이 약 3km, 전주 8개
- 자산: **modular_electricity_poles** (Poly Haven CC0)
- 결함: **3종** — 애자 균열·오손 / 변압기 누유 / 완금 부식
- 봇: 1~2대 (러버밴딩)
- 모드: 튜토리얼 + 챔피언십(싱글) + 타임어택
- 시야: 3인칭 + FPV 줌
- 이벤트: 전자파 간섭 1종 + 돌풍 1종
- 점수: 시간 + 점검 정확도 + 부스터 콤보
- 배포: Vercel 인터넷망

**검증 질문 (MVP 완료 후)**
1. 신규 점검자가 30분 플레이 후 결함 사진 식별이 더 정확? (사전/사후 테스트)
2. 30분 이상 자발적으로 플레이? (재미)
3. 베테랑이 "결함 시각화 현실적"이라 인정? (정성)
4. 카트라이더식 컨트롤이 드론에서 자연스러움? (UX)
5. 전사 표준 PC(HD 530)에서 30 FPS 이상? (성능)

→ 5개 모두 Yes여야 Phase 2 진입.
